# Channel Profiles — the Form Contracts

Load the target channel's section on every draft. Four fields per profile: **register** (tone), **length contract** (ceiling, not target), **structure**, **subject/title rules**.

## Contents

- Email — formal · family · work
- Text / SMS
- Teams / Slack
- GitHub release notes
- YouTube — titles + descriptions
- Social posts
- Discord announcements

---

## Email

**Formal** (institutions, strangers, disputes): register — courteous, complete sentences, zero slang; length — ≤200 words body; structure — greeting · one-line purpose · detail paragraph · explicit ask · sign-off; subject — 4–8 words, noun-led, no "quick question".
**Family**: register — warm, contractions fine, plain; length — ≤120 words; structure — lead with the point, detail only if asked-for; subject — plain words, or none if replying.
**Work** (colleagues, known contacts): register — direct professional, first names; length — ≤150 words; structure — purpose in line 1 · context · ask with a date; subject — verb-led, decision-relevant ("Approve Q3 deck by Fri").

## Text / SMS

Register — casual-direct, matches the relationship; length — ≤2 message bubbles, ≤300 chars total; structure — point first, logistics second, one question max; no subject. Never multi-topic.

## Teams / Slack

Register — professional-casual; emoji sparingly and only to aid scanning; length — ≤120 words or 5 bullet lines; structure — bold lead line · bullets for parallel facts · thread for detail; @mentions only for owners/deadlines, never @channel unless truly all-hands; no subject — the lead line is the subject.

## GitHub release notes

Register — factual, changelog-plain, second person for actions users take; length — ≤40 words per item; structure — Keep-a-Changelog buckets (Added / Changed / Fixed / etc.), breaking changes first and marked, upgrade note last; title — `vX.Y.Z — <3–6 word theme>`.

## YouTube

**Title:** ≤60 chars, front-load the payoff keyword, no clickbait promises the video can't cash. **Description:** first 2 lines carry the hook + core keywords (they show before the fold); then chapters/timestamps; then links; then boilerplate. Register — energetic but literal to the content.

## Social posts

Register — one idea per post, native to the platform's cadence; length — X/short-form ≤240 chars, LinkedIn ≤120 words; structure — hook line · substance · one CTA max; hashtags ≤3 and only load-bearing ones; no subject.

## Discord announcements

Register — community-warm, direct; length — ≤150 words; structure — `**bold headline**` · what changed / what's happening · what members should do · one link block; ping the narrowest role that covers the audience, @everyone only for genuinely-everyone news.
